🔥 Firebase Setup Guide

Follow these steps once and your QueueLess app will be fully live with a real-time database. Takes about 10 minutes.

1

Create a Firebase Project

Go to console.firebase.google.com and sign in with your Google account.

Click "Add Project" → Name it queless → Disable Google Analytics (not needed) → Click Create Project.

2

Enable Realtime Database

In the left sidebar, click Build → Realtime Database → Create Database.

Choose your region (pick the closest one). When asked about rules, select "Start in test mode" (we'll secure it later). Click Enable.

3

Get Your Firebase Config

Go to Project Settings (gear icon, top left) → Scroll down → Under "Your apps", click the </> Web icon.

Register the app (any nickname) → You'll see a config object like this:

const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "queless.firebaseapp.com", databaseURL: "https://queless-default-rtdb.firebaseio.com", projectId: "queless", storageBucket: "queless.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" };
⚠️ Copy this entire block — you need it in the next step.
4

Paste Config into Your Files

Open queue.html and admin.html in VS Code (or any text editor).

Find the line that says // ⚠️ REPLACE THIS with your Firebase config and replace the entire firebaseConfig object with the one you copied above.

Do this in both files.

5

Change the Admin Password

In admin.html, find this line:

const ADMIN_PASSWORD = "queless123"; // Change this!

Replace queless123 with your own password.

6

Push to GitHub & Go Live

Replace the files in your GitHub repo with the new ones. Your site should update automatically if you're using GitHub Pages.

If using Vercel or Netlify, just drag and drop the files — they deploy automatically.

🎉 You're Live!

Once done, open your site → Admin login → Add yourself to a test queue → Watch the live position update in real time.
